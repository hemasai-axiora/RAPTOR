## 1. Functional Requirements Overview
This document defines the functional requirements for Raptor CRM. It outlines the capabilities, behaviors, user flows, and role-based permissions required for the application.

---

## 2. User Authentication & Session Management

### 2.1 User Registration and Authentication
* The system shall allow users to register an account (Admin role only).
* The system shall authenticate users using secure credentials (email and hashed password).
* The system shall provide secure password reset capabilities via email validation.
* The system shall support secure session management, auto-logging users out after 30 minutes of inactivity.

### 2.2 Role-Based Redirects
Upon successful authentication, the system shall redirect users to their role-specific landing dashboard:
* **Admin:** Redirect to the *Executive Marketing Overview Dashboard* with full admin configurations enabled.
* **Manager:** Redirect to the *Executive Marketing Overview Dashboard* with campaign management controls.
* **Analyst:** Redirect to the *Channel & Campaign Performance Dashboard* to review marketing metrics.
* **Employer (Client):** Redirect to the *Executive Marketing Overview Dashboard* in a strict read-only mode.

## 3. Dashboard Functional Specifications

### 3.1 Executive Marketing Overview
The system shall display the Executive Dashboard to authorized roles (Employer, Manager, Admin). The dashboard must include:
* **Key Performance Indicators (KPIs):**
  * *Marketing Health Score:* A calculated index out of 100 based on campaign conversions, lead scores, and budget efficiency.
  * *Marketing ROI:* Calculated dynamically as `Revenue Influenced / Marketing Spend`.
  * *Revenue Influenced:* The total revenue linked to lead conversions.
  * *Qualified Leads:* The count of leads classified as "qualified" by managers.
  * *Lead Quality Score:* An index out of 100 based on the proportion of Hot vs. Cold leads.
  * *Customer Acquisition Cost (CAC):* Total campaign spend divided by the number of acquired customers.
  * *Conversion Rate:* The percentage of website visitors converted to leads.
  * *Customer Lifetime Value (LTV):* Average revenue generated per customer.
* **KPI Features:** Each KPI card must display the raw metric, status label (e.g., Healthy, Watch, Action), percentage growth vs. the previous 30-day period, and a sparkline chart showing daily trends.
* **Interactive Visualizations:**
  * *Spend vs. Revenue Chart:* A combination chart plotting monthly marketing spend (bar), revenue influenced (bar), and ROI (line).
  * *Marketing Funnel:* A vertical funnel displaying the absolute volume, stage-to-stage conversion rate, and period-over-period percentage growth for:
    `Reach -> Impressions -> Website Visitors -> Engaged Users -> Leads -> Qualified Leads -> Customers -> Revenue`
  * *Platform Performance:* A horizontal stacked bar chart comparing channels (Instagram, Facebook, LinkedIn, YouTube, X, Website) across Reach, Engagement, Leads, and Revenue.
  * *Campaigns Grid:* A table listing active campaigns with columns: Campaign Name, Spend, Leads, Revenue, ROI, and Status.
  * *AI Insights Panel:* A list of highlights generated by AI analyzing current performance and providing recommendations.
  * *Smart Alerts Banner:* A bottom-docked banner displaying real-time critical warnings (e.g. "ROI dropped on X (Twitter) by 18%").

### 3.2 Channel & Campaign Performance
The system shall display the Channel & Campaign Performance Dashboard to Analysts, Managers, and Admins. The dashboard must include:
* **Performance Comparisons:**
  * *Platform Table:* Comparative columns for Platform, Reach, Engagement Rate, Leads, Revenue, and ROI.
  * *Campaign Performance (Top 7):* A data grid showing active campaigns, spend, reach, leads, revenue, ROI, and status with inline pagination and filtering.
  * *Content Performance (Top 5):* A list of top-performing content assets, their publishing platforms, engagement rates, and reach.
* **Trend & Traffic Analyses:**
  * *Engagement Trend:* A dual-axis line/bar chart displaying daily total engagements (bars) vs. engagement rate (line).
  * *Traffic Sources:* A donut chart representing traffic acquisition channels (Organic Social, Direct, Paid Social, Referral, Email, etc.).
  * *Leads by Channel:* A horizontal bar chart illustrating leads attributed to each platform.
  * *Posting Heatmap:* A day-of-week vs. time-of-day grid visualizing average engagement rates to identify optimal publishing times.
  * *Budget Optimization Cards:* Interface recommendations for reallocating spend. If the user is a Manager or Admin, a button saying "Apply Recommendation" must execute the budget reallocation.

### 3.3 Customer Intelligence & Analytics
The system shall display the Customer Intelligence & Analytics Dashboard to Analysts, Managers, and Admins. The dashboard must include:
* **Customer Journey Map:** A visual flowchart showing the progression of contacts from `Reach` to `Customer`. It must calculate the drop-off percentage and total contact drop count between every consecutive stage.
* **Lead Quality Breakdown:** A donut chart indicating the percentage and count of Hot, Warm, and Cold leads.
* **Audience Demographics:** Charts tracking Age distribution, Gender distribution, Geographic performance (interactive world map), and Device usage.
* **Behavioral Analysis:**
  * *Web Overview:* Bounce Rate, Pages per Session, and Average Scroll Depth cards.
  * *Top Landing and Exit Pages:* Tables displaying pages with high traffic entrances and exit percentages.
  * *Multi-Touch Attribution:* A donut chart allowing the user to select from Attribution Models: First Touch, Last Touch, Data Driven, and Time Decay.
  * *Competitor Benchmarking:* A comparison matrix mapping Axiora Pulse metrics against industry benchmarks (Engagement, Growth, CTR, Conversion, Share of Voice).
  * *Sentiment Analysis:* A line chart displaying Positive, Neutral, and Negative brand sentiment trends over a rolling 30-day window.

## 4. Role Permissions & View Scopes
The system shall enforce strict data isolation and functional restrictions based on the active session role:

### 4.1 Employer (Client) Role Scope
* The system shall block access to:
  * Employee Management (profiles, attendance, task assignments).
  * Financial Setup (revenue tracking, invoice templates, billing settings).
  * CRM Database Settings (SMTP, backups, APIs, integrations).
  * Campaigns & Channels editing.
* The system shall render the *Executive Marketing Overview Dashboard* as read-only.
* The system shall hide all edit, delete, add, and configuration buttons.
* The system shall hide the left sidebar options: "Employee Management", "Settings", "Data Sources", and "Task Management".

### 4.2 Analyst Role Scope
* The system shall grant read-only access to all client records, campaign parameters, and lead details.
* The system shall block the Analyst from creating, editing, or deleting clients, campaigns, leads, or employee records.
* The system shall grant full write and configuration access to the *Reports* module, allowing Analysts to generate, export, print, and schedule reports.
* The system shall block access to the "Settings" and "Employee Attendance" views.

### 4.3 Manager Role Scope
* The system shall grant full create, read, update, and delete (CRUD) access for:
  * Client records, packages, and renewals.
  * Campaigns, budgets, and objectives.
  * Leads and lead assignments.
  * Content calendar drafts and publishing flows.
  * Employee profiles, tasks, and attendance tracking.
  * Invoices, payments, and expenses in the Finance module.
* The system shall block access to system-level settings, including SMTP settings, database backups, API credential setups, and user role creation.

### 4.4 Admin Role Scope
* The system shall grant unrestricted CRUD access to all modules, including system-wide configurations, database backups, SMTP, API integrations, and billing settings.
* The system shall allow the Admin to manage user accounts, assign user roles, and modify RBAC permissions.

## 5. Security & System Requirements
* **Password Encryption:** The system shall hash user passwords using `bcrypt` (minimum cost factor of 10) before saving them to the database.
* **SQL Injection Prevention:** The system shall use PDO prepared statements for all database operations.
* **Cross-Site Scripting (XSS) Prevention:** The system shall sanitize and escape all user-submitted data before rendering it in the browser.
* **Cross-Site Request Forgery (CSRF) Protection:** The system shall generate and validate unique CSRF tokens for all state-changing HTTP requests.
* **Audit Logging:** The system shall record an audit log in the `activity_logs` table for all critical changes (e.g. user logins, campaign budget updates, invoice creations, and setting alterations), capturing user ID, action, timestamp, and IP address.
